# Python 后端学习路线（前端开发者版）

## 你已经具备、可直接迁移的能力

| 前端经验 | 后端对应 |
| --- | --- |
| 调 REST API、axios | 自己设计并实现 REST API |
| TypeScript 类型 | Python Type Hints + Pydantic（和 TS 很像） |
| Vue Router 路由 | FastAPI 路由 / 路径参数 |
| Pinia 状态 | 服务端无状态，状态在 DB/Redis |
| 表单校验 | Pydantic 模型校验（类似 zod/yup） |
| JWT / Token 登录 | 后端签发与校验 JWT |
| 环境变量 `import.meta.env` | `.env` + python-dotenv |
| npm / package.json | pip / poetry / pyproject.toml |
| async/await | Python async/await（语法几乎一样） |

你不需要从零理解「什么是 API」，重点放在：**数据怎么存、权限怎么验、业务逻辑放哪、怎么部署**。

## 推荐技术栈（和你前端最顺）

**FastAPI + SQLAlchemy + PostgreSQL + Redis + JWT + pytest + Docker**

### 为什么选 FastAPI 而不是 Django

- 和 Vue 项目一样是前后端分离，只做 API
- 自动生成 OpenAPI/Swagger，你在前端已经习惯看接口文档
- Pydantic 模型和 TypeScript interface 思维一致
- 异步友好，性能不错，国内新项目用得多

Django 可以以后有需要再学（Admin 后台、复杂 CMS 时再考虑）。

## 8 周学习计划（有前端背景版）

### 第 1–2 周：Python 速成 + 环境

只学后端会用到的，不必啃完整 Python 书：

- 语法差异：`def`、`with`、列表推导、`dict` 用法
- 类型注解：`def get_user(id: int) -> User | None`
- 虚拟环境：`python -m venv .venv`，激活后 `pip install`
- 包管理：建议直接用 Poetry 或 uv（类似 npm）

**小练习：** 写脚本读 JSON 文件、过滤数据、输出新 JSON（模拟接口数据处理）。

### 第 3–4 周：FastAPI + 数据库

**目标：** 做一个和用户系统类似的 API，和你前端项目能对接。

**建议实现：**

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/auth/register` | 用户注册 |
| POST | `/api/auth/login` | 返回 JWT |
| GET | `/api/users/me` | 需要 Token |
| CRUD | `/api/items` | 列表、详情、创建、更新、删除 |

**技术点：**

- FastAPI 路由、`Depends` 依赖注入
- SQLAlchemy 2.0 模型 + Alembic 迁移
- PostgreSQL（或 MySQL，和前端项目保持一致更好）
- 统一响应格式：`{ code, data, message }`（很多 Vue 项目都这样）
- 分页：`page` / `page_size` 或 `offset` / `limit`

**对照前端：** 用你熟悉的 Vue 项目去调自己写的 API，立刻能感受 CORS、401、422 校验错误等。

### 第 5 周：认证、权限、Redis

- **JWT：** python-jose 或 PyJWT
- **密码：** bcrypt（永远不要明文存密码）
- **Redis：** 登录 Token 黑名单、简单缓存
- **角色权限：** 普通用户 / 管理员（对应你项目里的路由守卫）

**对照前端：** 前端 `Authorization: Bearer xxx` 怎么传，后端怎么解析 `Depends(get_current_user)`。

### 第 6 周：测试 + 工程规范

- pytest + FastAPI TestClient
- 测登录、测 CRUD、测无 Token 返回 401

**项目结构（推荐）：**

```
app/
  api/          # 路由（类似 views）
  schemas/      # Pydantic（类似 types）
  models/       # SQLAlchemy（数据库表）
  services/     # 业务逻辑
  core/         # 配置、安全、依赖
  main.py
```

这和 Vue 里 `views` / `types` / `composables` / `utils` 的分层思路一致。

### 第 7 周：文件上传、异步任务

结合你当前 AI 管理类项目，很实用：

- **文件上传：** 本地或 OSS（阿里云/MinIO）
- **Celery + Redis：** 耗时任务（如配音、模型调用）放队列
- **Webhook / 回调：** 异步通知前端或轮询状态

### 第 8 周：Docker 部署

- Dockerfile + docker-compose.yml（app + postgres + redis）
- Uvicorn 跑 FastAPI，前面 Nginx 反代
- 和前端一样：dev / prod 环境变量分离

跑通后：**Vue 前端 → 你的 Python API → 数据库**，整条链路就通了。

## 练手项目建议（贴近你现状）

做一个「AI 配音 / 模型管理」简化版后端：

| 模块 | 接口示例 |
| --- | --- |
| 用户 | 注册、登录、个人信息 |
| 模型配置 | CRUD（对应你 EditModelDialog 那类数据） |
| 配音任务 | 创建任务、查询状态、历史列表 |
| 文件 | 上传音频、返回 URL |

前端可以继续用现有 Vue 项目，把 mock 或旧接口逐步换成自己的 FastAPI——学以致用最快。

## 学习顺序：先别碰的

有前端背景时容易踩坑的地方：

- **不要先学 Django 全家桶** — 概念多，和「纯 API + Vue」不匹配
- **不要过早上 K8s / 微服务** — 先把单体 API + Docker 跑稳
- **不要跳过 SQL** — ORM 会写不等于懂索引、JOIN、事务
- **不要忽视 Python 虚拟环境** — 类似前端 node_modules，别全局乱装包

## 和 TypeScript 的对照（加速理解）

```typescript
// TypeScript
interface UserCreate {
  username: string
  password: string
}
```

```python
# Python + Pydantic（几乎一一对应）
class UserCreate(BaseModel):
    username: str
    password: str

# FastAPI 路由 ≈ Vue 里定义一个 API 模块
@router.get("/users/{user_id}", response_model=UserOut)
async def get_user(user_id: int, db: Session = Depends(get_db)):
    ...

# Depends ≈ 组合式函数 / 注入
async def get_current_user(token: str = Depends(oauth2_scheme)) -> User:
    ...
```
